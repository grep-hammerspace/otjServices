Plan: Encrypt Passwords at Rest

Context

User passwords are currently stored as plaintext in MongoDB. They need to be recoverable (not just verified) because they are used to drive Selenium browser login via OtjDriver.prepareBrowser. This rules out hashing — symmetric encryption (AES-256-GCM) is the right approach. The encryption key must come from an environment variable, not be hardcoded.

Approach

Introduce a single PasswordCipher class that owns all crypto logic. Inject it into UserRepository, which becomes the only layer that ever sees ciphertext. The rest of the app (OtjServicesResource, OtjDriver) continues to work with plaintext strings unchanged.

Step 1 — New file: src/main/java/com/github/grepHammerspace/crypto/PasswordCipher.java

  • Algorithm: AES/GCM/NoPadding, 256-bit key, 12-byte IV, 128-bit auth tag
  • No-arg constructor reads PASSWORD_ENCRYPTION_KEY env var (Base64-encoded 32 bytes); throws IllegalStateException on startup if missing
  • Second constructor accepting a Base64 key string directly — used by tests
  • Both constructors delegate to a private init(byte[]) method
  • Annotated @Singleton with @Inject on the no-arg constructor

encrypt(String plaintext) → String

byte[] iv = new byte[12];
new SecureRandom().nextBytes(iv);
Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
cipher.init(Cipher.ENCRYPT_MODE, key, new GCMParameterSpec(128, iv));
byte[] ciphertext = cipher.doFinal(plaintext.getBytes(StandardCharsets.UTF_8));
// prepend IV to ciphertext
byte[] combined = new byte[12 + ciphertext.length];
System.arraycopy(iv, 0, combined, 0, 12);
System.arraycopy(ciphertext, 0, combined, 12, ciphertext.length);
return "ENC:" + Base64.getEncoder().encodeToString(combined);

decrypt(String stored) → String

if (!stored.startsWith("ENC:")) return stored;  // migration passthrough for old plaintext docs
byte[] combined = Base64.getDecoder().decode(stored.substring(4));
byte[] iv        = Arrays.copyOfRange(combined, 0, 12);
byte[] ciphertext = Arrays.copyOfRange(combined, 12, combined.length);
Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
cipher.init(Cipher.DECRYPT_MODE, key, new GCMParameterSpec(128, iv));
return new String(cipher.doFinal(ciphertext), StandardCharsets.UTF_8);
All checked crypto exceptions should be caught and re-thrown as RuntimeException with a meaningful message.

Step 2 — Modify UserRepository.java

  • Add PasswordCipher passwordCipher as second constructor parameter
  • In save(): .append("password", passwordCipher.encrypt(user.password()))
  • In findByUserId(): wrap retrieved password with passwordCipher.decrypt(...) before constructing the User record

Step 3 — Modify AppModule.java

@Provides @Singleton
PasswordCipher providePasswordCipher() {
    return new PasswordCipher();  // reads PASSWORD_ENCRYPTION_KEY from env
}

Step 4 — Modify TestAppModule.java

@Provides @Singleton
PasswordCipher providePasswordCipher() {
    // 32 zero bytes — deterministic, test-only key
    return new PasswordCipher("AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=");
}

Step 5 — Modify UserRepositoryIT.java

Construct PasswordCipher with the same fixed test key and pass it to UserRepository. Plaintext assertions like assertEquals("secret", found.password()) continue to pass since findByUserId decrypts transparently. 

PasswordCipher cipher = new PasswordCipher("AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=");
repository = new UserRepository(database, cipher);

Step 6 — Modify registration.feature

The Cucumber step asserting password | pw1 against the raw MongoDB document will now see ENC:.... Replace with an assertion that the stored value starts with "ENC:", or remove the raw DB password check entirely.

Step 7 — Update docker-compose.yml and .env

Add to the app service environment in deploy/docker-compose.yml:

PASSWORD_ENCRYPTION_KEY: ${PASSWORD_ENCRYPTION_KEY}

Generate a key and add it to .env:

python3 -c "import secrets, base64; print(base64.b64encode(secrets.token_bytes(32)).decode())"
PASSWORD_ENCRYPTION_KEY=<generated-base64-key>

Migration

Existing plaintext passwords in MongoDB are handled by the decrypt() passthrough — if the stored value doesn't start with "ENC:" it is returned unchanged, so existing users can still log in. Their password is encrypted the next time they call POST /register.

Verification

  1. Generate a key and add it to .env
  2. Run mvn package — should compile cleanly
  3. Run mvn verify — UserRepositoryIT and Cucumber tests should pass
  4. Start the stack: docker compose -f deploy/docker-compose.yml up
  5. Call POST /otj-services/register with credentials
  6. Check MongoDB directly — the password field should contain ENC:...
  7. Call GET /otj-services/prepare-browser — Firefox should launch and navigate to the OTP page, confirming decryption works end-to-end
